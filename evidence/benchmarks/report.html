NVFP4 KV Cache — Tested

Status
Unsupported Architecture
Root cause: FlashInfer's TllmGenFmhaRunner does not support SM12.1 (GB10). The NVFP4 KV attention kernel requires SM90/SM100. Two mitigations applied:
  • Patched flashinfer.utils.canonicalize_torch_dtype to map "nvfp4" → torch.uint8
  • KV cache allocation succeeded · attention forward pass failed at kernel level
FP8 KV cache used for all benchmarks. NVFP4 KV requires a FlashInfer update for SM12.1.

Thermals & Power

GPU temperature rises from 48°C idle to 62°C peak under sustained c5 load. Power draw remains at ~22W per GPU — well within thermal limits. No throttling observed across any benchmark run.

Known Issues & Findings

NVFP4 KV Cache — Blocked
TllmGenFmhaRunner does not support SM12.1 (GB10). Requires FlashInfer update. Dtype mapping patched (canonicalize_torch_dtype), but kernel-level support absent.
GPU_UTIL > 0.85 — Engine Crash
Worker rank 0 dies after 3–5 min at 0.85 GPU util. Stable at 0.70. Model uses 58.58 GiB/node; insufficient headroom at 0.85 for KV + Ray + OS overhead.
vLLM v0.22 — Incompatible
Step3p7 custom model code has 5+ incompatibilities with v0.22's generic Transformers backend: no TP interface, wrong weight names, API mismatches. Native Step3p7 support required.
