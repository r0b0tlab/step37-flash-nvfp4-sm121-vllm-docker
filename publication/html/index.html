Launch recipe

vllm serve stepfun-ai/Step-3.7-Flash-NVFP4 \
  --served-model-name step3p7 \
  --tensor-parallel-size 2 \
  --distributed-executor-backend ray \
  --disable-custom-all-reduce \
  --quantization modelopt \
  --kv-cache-dtype fp8 \
  --disable-cascade-attn \
  --gpu-memory-utilization 0.90 \
  --max-model-len 65536 \
  --max-num-batched-tokens 65536 \
  --max-num-seqs 5 \
  --reasoning-parser step3p5 \
  --tool-call-parser step3p5 \
  --enable-auto-tool-choice \
  --enforce-eager

Network: NCCL_SOCKET_IFNAME, GLOO_SOCKET_IFNAME, TP_SOCKET_IFNAME must point to QSFP interface

Known limitations

  • CUDA graphs at 64K+ context cause SHM broadcast deadlock on cross-node TP=2
  • --enforce-eager required on first run (Triton JIT + CUDA graphs cause RPC timeout)
  • Upstream recommends TP=4+EP; TP=2 is our validated target for dual GB10
  • PYNCCL all-reduce (SM121 NCCL_SYMM_MEM unavailable)
  • nsys cannot trace Ray worker GPU kernels across distributed nodes
